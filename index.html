Anatomie d'un bon prompt : penser CRTF

Un prompt efficace précise ces quatre éléments pour guider l'IA sans ambiguïté.

Contexte

Expose le cadre, les données disponibles et les contraintes pour que l'IA sache « où elle est ».

Rôle

Indique le persona attendu (enseignant, analyste, coach…) afin d’obtenir le ton et l’angle adéquats.

Tâche

Décrit clairement l’objectif à atteindre, les étapes clés ou les critères de réussite.

Format

Précise la forme finale : tableau, plan, liste numérotée, script… voire la longueur souhaitée.

Mini-procédure : de l'idée à l'activité interactive

  1. Étape 1 · Préparer un prompt CRTF pour le LLM

    Contexte : Tu aides un formateur à concevoir une activité courte pour tableau numérique en classe de français.

    Rôle : Agis comme un développeur pédagogique spécialisé en productions HTML/JS simples.

    Tâche : Génère un seul fichier combinant HTML et JavaScript qui affiche des phrases en français ; chaque verbe change de couleur au survol du curseur pour attirer l’attention des apprenants.

    Format : Fournis le code complet dans un seul bloc, prêt à être collé tel quel dans un fichier `.html`, avec des commentaires brefs expliquant où se trouvent les verbes et la logique de survol.

    Prompt à envoyer : « Conçois un unique fichier HTML incluant son script JS inline... » (adapter en fonction du niveau).

  2. Étape 2 · Copier et sauvegarder le fichier

    • Copier l’intégralité du code fourni par le LLM.
    • Ouvrir Bloc-notes ou VS Code, coller le texte sans modification.
    • Enregistrer sous, choisir « Tous les fichiers », nommer par exemple `verbes-survol.html`, encodage UTF-8.

  3. Étape 3 · Déployer sur le tableau numérique

    • Transférer le fichier sur la clé USB ou dans le cloud accessible depuis la salle.
    • Ouvrir le fichier `.html` dans un navigateur (Chrome/Edge) depuis le tableau numérique.
    • Tester le survol des verbes pour vérifier l’effet de couleur avant l’activité.

Ajouter un quiz sur ELEA à partir d'un fichier AIZEN

Clarification : un fichier AIZEN remplit la banque de questions Moodle/ELEA. Ce n’est pas un contenu H5P. Après import, vous devez créer l’activité standard « Test » pour servir de contenant.

1 · Activer le mode édition

Vérifiez que le bouton « Activer le mode édition » est enclenché dans votre cours ELEA. Si « Quitter le mode édition » s’affiche, c’est déjà actif.

2 · Ajouter l’activité « Test »

Dans la section souhaitée, cliquez sur « Ajouter une activité ou une ressource », choisissez l’icône Test (). Donnez un titre clair (ex. « Quiz de révision - Méthodologie ») puis « Enregistrer et afficher ».

3 · Paramétrer rapidement le contenant

Laissez les réglages avancés (dates, tentatives, temps) pour plus tard si besoin. L’objectif est d’obtenir le squelette du quiz vide.

4 · Ajouter les questions importées

Sur la page du Test, cliquez sur « Modifier le test » puis sur le menu « Ajouter » > « ... de la banque de questions ». Sélectionnez les items issus du fichier AIZEN généré par votre requête IA et importé précédemment, puis « Ajouter les questions sélectionnées ».

5 · Finaliser et publier

Ajustez la note maximale si besoin (10, 20, etc.), cliquez sur « Enregistrer ». Revenez à la page du cours : le Test est visible et prêt pour les étudiants.

Besoin spécifique IA → AIZEN ?

Rédigez un prompt CRTF demandant au LLM de produire un paquet de questions au format GIFT ou XML Moodle, puis convertissez-le en AIZEN via l’outil ELEA prévu. Importez dans la banque de questions avant d’appliquer les étapes ci-dessus.
